# 문서 작성 규칙 (Doc Writing Rules)

Claude가 이 저장소에서 문서를 작성하거나 수정할 때 반드시 따라야 할 규칙입니다.

---

## 1. 언어 및 문체

- **기본 언어**: 한국어
- **영어 원문 병기**: 기술 용어는 첫 등장 시 반드시 영어 원문 병기
  - 예: 파드 (Pod), 디플로이먼트 (Deployment), 레플리카셋 (ReplicaSet)
- **문체**: 명사형 종결 (`~입니다` 대신 `~함`, `~사용`)
- **존댓말 금지**: 기술 문서 특성상 경어체 사용하지 않음

## 2. 코드 블록

- **언어 태그 필수**: 모든 코드 블록에 언어 지정
  - ` ```yaml ` — Kubernetes 매니페스트
  - ` ```bash ` — shell/kubectl commands
  - ` ```go ` — Go 코드
  - ` ```json ` — JSON configs
- **실행 가능한 코드**: 복붙해서 바로 실행되는 수준으로 작성
- **플레이스홀더 표시**: 교체 필요한 값은 `<YOUR_VALUE>` 형식으로 명시

## 3. 섹션 구성 필수 요소

새 문서는 반드시 아래 섹션을 포함해야 함:

| 섹션 | 내용 | 생략 가능 여부 |
|------|------|----------------|
| 1. 개요 및 비유 | 정의 + 직관적 비유 | 불가 |
| 2. 핵심 설명 | 원리, YAML, Best Practice | 불가 |
| 3. 트러블슈팅 | 증상 → 원인 → 해결 | 불가 |
| 4. 모니터링 및 확인 | kubectl 확인 명령어 | 불가 |
| 5. TIP | 현장 팁 | 가능 |

## 4. 비유 작성 원칙

- 💡 비유 섹션: 초보자도 이해할 수 있는 일상적인 비유 사용
- 예: "Pod는 컨테이너의 기숙사", "Deployment는 Pod의 관리 감독관"

## 5. 트러블슈팅 작성 형식

```markdown
### {증상 이름}

**증상**: 어떤 오류 메시지가 나타나는지

**원인**: 왜 발생하는지 근본 원인

**해결 방법**:
```bash
kubectl ...
```
```

## 6. 금지 사항

- 추측성 내용 작성 금지 (실제 경험/공식 문서 기반만)
- "~할 수 있습니다", "~일 것입니다" 같은 불확실한 표현 금지
- YAML 일부만 발췌해서 컨텍스트 없이 제시 금지 (완전한 스펙 제공)
